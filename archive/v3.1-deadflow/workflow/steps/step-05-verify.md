# Step 05 — 验证

## MANDATORY EXECUTION RULES
- 🛑 质量 < 60 必须回退到 step-04
- 📋 烛自动选评审模式（不全用 standard）

## 验证流程

1. 代码变更 → 召唤烛（自动选模式）
2. 嵌入式相关 → 召唤匠验证
3. 读下游建议 → 链式验证（上限 2 层）

## 质量评分（100 分制）
| 维度 | 权重 |
|------|------|
| 需求满足度 | 30 |
| 产物质量 | 25 |
| 副作用控制 | 20 |
| 完整性 | 15 |
| 可维护性 | 10 |

## 门禁
- ≥ 80 → 自动进 step-06
- 60-79 → 提示问题，用户决定
- < 60 → ⛔ 回退 step-04
