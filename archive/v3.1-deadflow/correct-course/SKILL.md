# Correct Course — 偏离修正

> 偏离是正常的。重要的是及时识别和修正。

## 触发场景

- 实施中发现原计划不可行
- 新信息导致需要调整方向
- 质量门禁未通过需要修正
- 主人说"偏了"或"不对"

## 修正流程

### Step 1：加载当前状态
读取：
- 原计划（handoff 中的 workflow-plan 或 spec）
- 当前进展（已完成的任务、已产出的文件）
- 偏离信号（用户反馈 / 测试失败 / 质量问题）

### Step 2：评估偏离
```markdown
## 偏离评估
- **原计划目标**：{一句话}
- **当前实际状态**：{一句话}
- **偏离程度**：轻微 / 中等 / 严重
- **偏离原因**：{新发现 / 假设失效 / 需求变更 / 技术障碍}
```

### Step 3：提出修正方案
```markdown
## 修正方案
### 方案 A：{调整路径继续}
- 改什么：
- 影响范围：
- 预计额外工作：

### 方案 B：{回退到某节点重做}
- 回退到哪：
- 需要重做什么：
- 预计额外工作：
```

### Step 4：落盘
路径：`.ai-shared/handoff/correct-course__{topic}__{YYYYMMDD-HHmm}.md`

### Step 5：等用户决定
展示方案，等主人选择。不自动执行修正。

## 与 workflow 的关系

workflow Phase 3 执行中遇到问题 → 触发 correct-course → 修正后回到 Phase 3 继续。

## 约束

不自动执行修正方案。偏离评估必须基于事实（引用具体文件/产物）。
