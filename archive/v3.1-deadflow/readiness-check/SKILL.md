# Readiness Check — 实现就绪门控

> 在写第一行代码之前，确认一切准备就绪。

## 检查维度

### 1. 规格完整性
- [ ] 目标明确（一句话可描述）
- [ ] 验收标准可测试
- [ ] Out-of-scope 已明确
- [ ] 硬约束已列出

### 2. 架构对齐
- [ ] 技术选型已确定
- [ ] 依赖已识别
- [ ] 接口已定义
- [ ] 与现有代码兼容性已评估

### 3. 任务树就绪
- [ ] 任务分解到可执行粒度
- [ ] 依赖关系清晰（DAG）
- [ ] 负责角色已分配
- [ ] 估时合理

### 4. 风险缓解
- [ ] 关键风险已识别
- [ ] 缓解措施已设计
- [ ] 回退方案已准备

### 5. 上下文完备
- [ ] context.md 已更新
- [ ] 相关 decisions 已记录
- [ ] 前序调研产物可引用

## 输出

```markdown
## 实现就绪评估

### 状态：[READY / NEEDS WORK / NOT READY]

### 各维度评分
| 维度 | 状态 | 缺口 |
|------|------|------|

### 关键缺口（如有）
1. {具体缺什么}

### 推荐下一步
1. {具体行动}
```

## HALT 条件

NEEDS WORK / NOT READY 时必须列出**具体**的缺口和修复动作。
不允许笼统说"需要更多信息"。
