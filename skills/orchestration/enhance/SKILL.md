# Enhance — Prompt 增强

> 把模糊变清晰，把口语变可执行。

## 增强流程

### Step 1：分析原始输入
沉默分析 `$ARGUMENTS`，识别：意图、缺失信息、隐含假设、项目上下文线索。

### Step 2：结构化补全（5 字段）

```markdown
## 增强后需求
### 1. 明确目标（一句话）
### 2. 技术约束（语言/框架/平台）
### 3. 范围边界（做什么 / 不做什么）
### 4. 验收标准（可测试条件 checkbox）
### 5. 相关上下文（文件/模块/先前决策）
```

### Step 3：展示对比
原始需求 vs 增强版本，用 AskUserQuestion 让用户选择。

### Step 4：输出
独立调用 → 展示结果。被 workflow/auto-pilot 调用 → 传递给下一阶段。

## 增强原则

1. **补全而非改变**——保留用户原意
2. **具体而非泛化**——用文件名/函数名替代模糊描述
3. **简洁而非冗长**——精炼不堆砌
4. **可执行而非描述**——输出是任务不是文档
5. **项目感知**——读 CLAUDE.md/context.md 获取真实上下文
