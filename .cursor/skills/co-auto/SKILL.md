# Auto Pilot — 全自动编排

> 说一句话，系统自己判断、自己调度、自己执行、自己验证。

## 自动驾驶规约

1. **最大自治**——默认不中断，用户说完就等结果
2. **静默增强**——Prompt 增强在内部完成，不展示不等确认
3. **自动选策略**——根据复杂度自动选择执行路径
4. **自动过门禁**——质量 ≥ 60 自动通过
5. **仅在必要时中断**：守卫层命中 / 质量 < 60 / 意图不清 / 等价方案

## Phase A：智能分析（静默）

1. 静默增强 `$ARGUMENTS`（读 context.md + decisions.md）
2. 复杂度评估：低(1-3) / 中(4-6) / 高(7-10)
3. 双表扫描——Subagent 表（烛/织/匠/策/鉴）+ Skill 表（keil/gcc/jlink/...）
4. 三路判定：Skill 直调 / Subagent 召唤 / 主驾直达 / 串联
5. **白发预检**（防过度发火·仅 🟡 路由）：若本轮要召唤 subagent 且属 🟡 档，先查最近同类发火的 `__DELTA__` 记录——**某类 🟡 路由近期持续零增量（DELTA=0）→ 本轮默认降级为主驾直达**，一句话告知"此类路由近期零增量，本轮改直达，如需仍可手动召唤"。**铁律边界**：只降 🟡；🔴 档（安全/生产前/外部实时事实）永不自动降级（守 rules.md §三铁律1）。**降级保险**：DELTA 账本为空（数据未积累）→ 静默跳过、不阻塞、退回常规判定。

## Phase B：策略选择（自动）

| 复杂度 | 策略 |
|--------|------|
| 低 1-3 | 🚀 直达——主驾直接执行 |
| 中 4-6 | 🎯 精准调度——单 agent/skill 定点打击 + 链式下游 |
| 高 7-10 | 🎭 Party 并行——spawn 2-4 独立 agent 多视角（party-mode，随时可用）；超大/需确定性 fan-out → harness **Workflow 工具**（多 agent 并行 + 对抗验证 + 可复现，需 ultracode 或主人显式授权） |

## Phase C：自动执行

**🚀 直达**：Read/Edit/Bash 直接做。
**🎯 精准调度**：
1. 选最匹配 agent（烛自动选评审模式）
2. 静默准备任务卡
3. 召唤 → 读下游建议 → 链式调度（上限 3 层）
4. ≥ 60 分综合输出 / < 60 重试

**🎭 Party 并行**：复杂问题 spawn 2-4 独立 agent 多视角讨论（真思维多样性），主驾综合。需确定性 fan-out 的超大任务 → harness **Workflow 工具**（脚本化多 agent 编排 + 对抗红队 + 可复现；v3.3 深度完善的 42-agent 并行测绘即实战锚，见 `514cc/.ai-shared/handoff/synthesis__deep-evolution-v33__*`）。注意：Workflow 工具需 ultracode 开启或主人显式授权才可发起（成本高，非默认）。
